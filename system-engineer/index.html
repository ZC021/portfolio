Linux 서버 이상 징후 모니터링 자동화

사용자별 원격 작업 지연을 확인하고, AWX·Ansible 반복 점검과 대시보드로 영향 범위를 비교했습니다.

2026.02–2026.0633개 Linux 서버30분 주기

문제와 최초 확인

  • 일부 사용자 계정에서 SSH 로그인 이후 작업과 VSCode 원격 세션이 지연되거나 정상적으로 이어지지 않는 문제 발생
  • 관리자 계정으로 SSH 접속한 뒤 ps로 대상 사용자 프로세스를 확인해 입출력 대기 상태(D-state)가 다수 존재하는 것을 확인

원인 범위 확인

  • wchan과 실행 작업을 함께 확인해 NFS 홈·공유 경로를 사용하는 프로세스가 반복적으로 포함된 것을 확인
  • git status와 VSCode 서버 등 파일 조회가 많은 작업을 우선 확인 대상으로 분류
rpc_wait_bit_killable nfs_wait_on_request lookup_slow rwsem_down_write_slowpath
클라이언트 설정 검토 제한된 범위에서 NFS hard·soft 설정을 각각 테스트했지만 동일한 증상이 나타났습니다. 마운트 옵션 변경만으로 해결하기 어렵다고 판단해 공유 NFS /home을 분리하는 방향으로 다음 테스트를 진행했습니다.

확인에서 반복 점검까지

  1. 사용자별 SSH·VSCode 작업 지연
  2. ps로 대상 사용자 확인
  3. wchan·작업 경로 대조
  4. hard·soft 테스트
  5. AWX·Ansible 30분 점검
  6. 서버별 대시보드 비교

자동화 구성

  • 운영 데이터를 변경하지 않는 상태 확인 명령을 Ansible 작업으로 구성
  • AWX에서 33개 Linux 서버를 대상으로 30분마다 실행
  • 서버별 D-state 수·wchan·프로세스·사용자 현황을 비교하는 대시보드 구성
  • AI 도구와 협업해 점검 대시보드를 구현하고, 점검 대상·주기·수집 항목과 실제 결과는 직접 검증

판단 기준

  • D-state 수만으로 단일 원인을 확정하지 않음
  • 반복 점검 결과와 실행 작업·접근 경로를 함께 비교
  • 영향이 집중된 사용자·서버를 다음 제한 테스트 대상으로 결정

결과와 다음 단계

  • 서버별 수동 확인을 30분 주기 자동 점검과 대시보드 비교로 전환
  • NFS 경로 사용 작업과 이상 징후가 함께 나타난 사용자·서버를 우선 확인 대상으로 분류
  • 설정 변경으로 증상이 개선되지 않은 결과를 바탕으로 iSCSI 제한 테스트와 /home 분리 검토로 연결

FC SAN 전환 프로젝트 참여

NFS 설정 변경으로 개선되지 않은 사용자별 작업 지연을 다른 블록 스토리지 경로에서 확인한 뒤 FC SAN 전환에 참여했습니다.

2026.02–2026.06Dell PowerStore 3200TiSCSI 제한 테스트 → FC SAN

검증 배경

  • 반복 점검과 hard·soft 테스트에서도 기존 증상이 동일해 공유 NFS /home을 다음 검증 대상으로 설정
  • 운영 전환 전에 일부 사용자 범위에서 저장 경로를 분리했을 때 기존 지연이 다시 나타나는지 확인할 필요

iSCSI 제한 테스트

  • 기존 Ethernet 환경을 활용할 수 있는 iSCSI로 Dell PowerStore 3200T를 일부 서버에 연결
  • 테스트 기간과 해당 사용자 범위에서는 기존 SSH·VSCode 작업 지연이 관찰되지 않음을 확인

검증에서 전환까지

  1. NFS 설정 변경 후 동일 증상
  2. /home 분리 가설
  3. 일부 사용자 iSCSI 테스트
  4. 테스트 범위에서 지연 미관찰
  5. FC SAN 전환
  6. /home 데이터 전환

FC SAN 전환 참여

  • 운영 환경은 업무용 Ethernet과 스토리지 트래픽을 별도 경로로 분리하는 요건에 따라 FC SAN으로 결정
  • 주말 작업에서 서버 종료 후 FC HBA 장착과 광케이블 연결 수행
  • 서버 측 연결·스토리지 인식·마운트 상태 확인

데이터 전환과 협업 범위

  • rsync를 이용한 기존 NFS /home 데이터 전환과 확인에 참여
  • 여러 팀이 공유해야 하는 /team은 NFS로 유지
  • 스토리지와 SAN 스위치 내부 설정은 전문 엔지니어가 담당

결과

  • 테스트 기간과 사용자 범위에서 기존 SSH·VSCode 작업 지연이 관찰되지 않음을 확인
  • 개인 /home의 NFS 의존도를 줄이고, 공유가 필요한 /team은 기존 NFS에 유지
